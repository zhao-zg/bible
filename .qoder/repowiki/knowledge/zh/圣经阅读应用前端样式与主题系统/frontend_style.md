该应用采用**原生 CSS + CSS 变量（Design Tokens）**的轻量级样式架构，不依赖 Tailwind、Bootstrap 等第三方 UI 框架。核心设计围绕**多主题切换**、**响应式布局**及**PWA/原生混合环境适配**展开。

### 1. 样式架构与方法论
- **CSS 变量驱动**：所有颜色、间距、阴影均通过 `:root` 和 `[data-theme]` 属性选择器定义的 CSS 变量管理。这种设计使得主题切换无需替换类名或重载页面，仅需修改根元素的 `data-theme` 属性即可实时生效。
- **模块化 CSS**：样式分为两个主要文件：
  - `style.css`：全局通用样式，包含布局、组件（卡片、按钮、导航）、动画及基础重置。
  - `bible-theme.css`：专为圣经阅读场景设计的主题样式，涵盖经文排版、注脚、串珠引用及阅读工具栏。
- **无预处理器**：直接使用原生 CSS，通过注释分块（如 `/* ── 一、CSS 变量 */`）组织代码，便于维护。

### 2. 主题系统 (Theming)
应用支持两套并行的主题体系，并通过 `theme-toggle.js` 进行统一管理和持久化（localStorage）：
- **阅读主题（5种）**：专为长时间阅读优化，包括 `gray-white`（灰白）、`light-yellow`（浅黄）、`warm-yellow`（米黄/暖色）、`dark-gray`（深灰）、`night`（黑夜）。
  - **暖色模式特色**：模拟纸张质感，使用奶油底色 (`#FBF7EF`) 和深棕文字 (`#4A3525`)，并对进度条、按钮激活态做了特殊适配以避免视觉疲劳。
- **界面主题（3种）**：`cool`（冷色/默认）、`warm`（暖色）、`dark`（夜间）。在初始化时会自动映射到对应的阅读主题（如 `cool` -> `gray-white`）。
- **系统跟随**：支持监听 `prefers-color-scheme`，在未手动设置主题时自动跟随系统深浅色。

### 3. 关键设计决策与规范
- **移动端优先与 PWA 适配**：
  - 使用 `env(safe-area-inset-*)` 处理 iPhone X+ 的刘海屏和安全区域。
  - 禁用默认滚动条 (`scrollbar-width: none`) 以提供更原生的 App 体验。
  - 底部控制栏（朗读、进度）固定定位，并针对小屏幕（<480px）做了字体和间距的微调。
- **交互反馈**：
  - 所有可点击元素均设置了 `-webkit-tap-highlight-color: transparent` 以移除移动端默认的点击高亮。
  - 使用 `touch-action: manipulation` 优化触摸响应，防止双击缩放延迟。
- **字体控制**：
  - 提供 5 级字号调节（14px - 22px），通过 JS 动态修改根字体大小或特定容器的 `font-size`。
  - 朗读速度支持 0.5x - 2.0x 调节，UI 上通过滑块实现。

### 4. 开发者约束
- **主题扩展**：新增主题需在 `style.css` 和 `bible-theme.css` 中同时定义完整的 CSS 变量集，并在 `theme-toggle.js` 的 `VALID_THEMES` 数组中注册。
- **样式隔离**：由于是静态站点生成（SSG），避免使用全局 ID 选择器，优先使用类名。组件样式（如弹框、Toast）通常附带特定的前缀（如 `cx-`）以防止冲突。
- **深色模式适配**：在编写新组件样式时，必须检查其在 `dark` 和 `night` 主题下的对比度，必要时使用 `[data-theme="dark"]` 选择器进行覆盖。
